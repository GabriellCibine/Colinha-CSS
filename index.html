Colinha Para Estudos De Css

Box Model

Para que serve:

Define como os elementos são renderizados na página, considerando conteúdo, padding, border e margin. Ou seja, tudo que ocupa espaço na tela.

O que precisa para usar:

Nada especial, o box model já é padrão do CSS.Mas você precisa entender que um elemento ocupa:

  • content → conteúdo (texto, imagem…)
  • padding → espaço entre conteúdo e borda
  • border → a borda em si
  • margin → espaço externo entre outros elementos

O que colocar nos filhos se usar no pai:

Não precisa colocar nada nos filhos para o box model funcionar, mas filhos também têm box model próprio. Às vezes usamos:

" box-sizing: border-box; "

no pai (ou globalmente) para os filhos não “quebrarem” o layout somando padding/border ao tamanho.

css:

* {
   box-sizing: border-box;
}

.box {
   width: 200px;
   height: 100px;
   padding: 20px;
   border: 5px solid black;
   margin: 10px;
}

html:
< div class="box" > Olá! < /div >

Quando e como usar:

Sempre! É fundamental entender o box model para controlar bem tamanhos e espaçamentos.

Display

Para que serve:

Define como um elemento é exibido na tela: bloco, inline, flex, grid, etc.

O que precisa para usar:

Colocar a propriedade display no elemento pai ou individualmente em cada elemento.

O que colocar nos filhos se usar no pai: Depende do tipo:

  • display: flex ou grid → filhos viram flex/grid items automaticamente.
  • display: block → cada filho quebra linha.
  • display: inline → filhos ficam na mesma linha.
  • display: inline-block → filhos ficam na mesma linha, mas aceitam tamanho.

Exemplo de uso:

css:

.container {
   display: flex;
}
.item {
   margin: 10px;
}

< div class="container" >
    < div class="item" > A </div >
    < div class="item" > B </div >
< /div >

Quando e como usar:

Quando quiser mudar o comportamento de empilhamento/alinhamento:

  • bloco → ocupa linha toda inline → ocupa só o necessário flex/grid → para layouts avançados

Flexbox

Para que serve:

Alinhar e distribuir elementos em linha ou coluna, com muito controle.

O que precisa para usar:

No pai:

css:

display: flex;

O que colocar nos filhos:

  • flex-grow, flex-shrink, flex-basis → para controle do espaço
  • order → muda a ordem visual
  • align-self → alinha individualmente

Exemplo de uso:

css:

.container {
   display: flex;
   justify-content: space-between;
   align-items: center;
}
.item {
   flex: 1;
}

html:

< div class="container" >
    < div class="item" > A < /div >
    < div class="item" > B < /div >
    < div class="item" > C < /div >
< /div >

Quando e como usar:

Quando quiser:

  • alinhamento fácil (horizontal/vertical)
  • distribuição automática de espaço
  • fazer menus, cards, cabeçalhos, etc.

Grid Layout

Para que serve:

Criar layouts em grade (linhas e colunas), perfeito para layouts de página.

O que precisa para usar:

No pai:

css:

display: grid;
grid-template-columns: repeat(3, 1fr);

O que colocar nos filhos:

  • grid-column, grid-row → para posicionar em lugares específicos
  • Ou deixar o grid auto-organizar

Exemplo de uso:

css:

.container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 10px;
}
.item {
    background: lightgray;
}

html:

< div class="container" >     < div class="item">1 < /div >     < div class="item">2 < /div >     < div class="item">3 < /div > < /div >

Quando e como usar:

Quando precisar de:

  • tabelas de cards
  • layouts de página com várias colunas/linhas
  • áreas responsivas e organizadas

Position

Para que serve:

Controlar o posicionamento exato dos elementos.

O que precisa para usar:

No elemento (pai ou filho):

position: relative | absolute | fixed | sticky;

O que colocar nos filhos se usar no pai:

Se o pai tem position: relative → filhos com position: absolute se posicionam relativos a ele.

css:

.container {
    position: relative;
}
.item {
    position: absolute;
    top: 10px;
    left: 20px;
}

html:

< div class="container" >     < div class="item" > Sou absoluto! < /div > < /div >

Quando e como usar:

Quando quiser:

  • criar sobreposição (ex.: modal, tooltip)
  • posicionamento fixo na tela (menu fixo, botão flutuante)
  • sticky headers

Propriedades Avançadas

Para que serve:

Aprimorar estilos com efeitos visuais e interações.

O que precisa para usar:

Só aplicar no CSS.

Exemplos:

  • box-shadow → sombras
  • transition → animações suaves
  • transform → rotação, escala
  • z-index → ordem de sobreposição

O que colocar nos filhos:

Depende: às vezes filhos precisam de z-index para não ficarem escondidos.

Exemplo de uso:

.button {
    background: blue;
    transition: background 0.3s;
}
.button:hover {
    background: red;
}

Quando e como usar:

Para melhorar experiência visual:

  • hover effects
  • animações
  • sombras, sobreposição

Refatoração

Para que serve:

Melhorar, simplificar e organizar o código CSS.

O que precisa para usar:

Nada especial, é prática de organização:

  • usar classes reutilizáveis
  • remover código duplicado
  • usar variáveis CSS (:root { --main-color: blue; })

O que colocar nos filhos:

Nada obrigatório, mas filhos devem herdar classes/estilos organizados.

Exemplo de uso:

css:

.button {
    padding: 10px;
    border-radius: 5px;
}
.primary {
    background: blue;
}
.secondary {
    background: gray;
}

Quando e como usar:

Sempre que o CSS crescer muito ou ficar confuso.

Responsividade

Para que serve:

Adaptar o layout para diferentes tamanhos de tela (celular, tablet, desktop).

O que precisa para usar:

Media queries:

css:

@media (max-width: 600px) {
   .container {
      flex-direction: column;
   }
}

O que colocar nos filhos:

Às vezes ajustar tamanho, padding, margem, ordem, etc., dentro da media query.

Exemplo de uso:

css:

.container {
      display: flex;
}
@media (max-width: 600px) {
   .container {
      flex-direction: column;
   }
}

Quando e como usar:

Quando quiser que o site funcione bem em celular e desktop.
